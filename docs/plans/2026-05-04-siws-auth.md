# SIWS Auth + Wallet Allowlist Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Replace the shared `WEB_CHANNEL_TOKEN` with Sign-In-With-Solana (SIWS) wallet-signature auth, gated by a mode-aware allowlist (owner / explicit list / open).

**Architecture:** The WebSocket connection accepts every client at handshake (origin still validated), then enters an `unauthenticated` state. The server issues a single-use nonce inside an `auth_challenge` message; the client signs a canonical SIWS message with their Solana wallet and returns `auth_response`. The server verifies the Ed25519 signature, checks the wallet against the configured allowlist tier, and either transitions to `authenticated` (chat enabled) or closes with `4001`. The connected, authenticated wallet IS the session's `walletAddress` — `wallet_connect` is folded into the auth handshake, eliminating the unverified self-assertion in the current protocol.

**Tech Stack:** Node 20 built-in `node:test` runner (no new test framework dep), `@noble/curves/ed25519` (already a transitive dep of Umi) or `tweetnacl` for signature verification, existing `bs58`, `crypto.randomBytes` for nonces, existing `zod` for config validation, existing Umi for `publicKey` parsing.

## Wire protocol (target state)

### 1. Connection

Client opens `ws://host:port/` (no `?token=`, no `Sec-WebSocket-Protocol: bearer`). Origin is still validated against `WS_ALLOWED_ORIGINS`.

### 2. Server → `auth_challenge`

Sent immediately after `connected`:

```json
{
  "type": "auth_challenge",
  "nonce": "9f4e7c1a2b...",                         // 32 hex chars (16 random bytes)
  "issuedAt": "2026-05-04T19:00:00.000Z",
  "expiresAt": "2026-05-04T19:01:00.000Z",          // issuedAt + AUTH_NONCE_TTL_MS
  "agentName": "Treasury Bot",
  "agentAsset": "ARsZN4kZUWNX8Ek5ZkVUpRitSFjym6p9jLvUxrg9kPWU",  // null if unregistered
  "network": "solana-mainnet",                      // or solana-devnet
  "authMode": "allowlist"                           // owner | allowlist | open
}
```

The client has `AUTH_NONCE_TTL_MS` (default 60s) to respond. After expiry the server closes with `4001 / nonce_expired`.

### 3. Canonical SIWS message

The client constructs **exactly** this UTF-8 string and signs its bytes:

```
Sign in to {agentName}

Agent: {agentAsset || 'unregistered'}
Network: {network}
Nonce: {nonce}
Issued: {issuedAt}
Expires: {expiresAt}
```

Wallets like Phantom/Solflare display this verbatim in the popup. Including `agentAsset` and `network` makes a signature non-replayable across agents and chains.

### 4. Client → `auth_response`

```json
{
  "type": "auth_response",
  "publicKey": "DwLLgBwG3eVNqHnp9CPsRKL2dVePVdXJq1tvNHCMC2YS",
  "signature": "5x9...7Qa",                         // base58, 64-byte Ed25519 sig
  "message": "Sign in to Treasury Bot\n\nAgent: ..."  // exact bytes the client signed
}
```

The server **re-derives** the canonical message from the stored nonce and compares to `message` byte-for-byte before verification. Mismatch → `4001 / message_mismatch`. This prevents a malicious client from signing a different string than what the wallet displayed (preserves WYSIWYG signing).

### 5. Server: verify

In order, fail-fast:

1. Nonce exists in store and not expired → else `nonce_invalid`
2. Provided `message` matches server-derived canonical → else `message_mismatch`
3. `nacl.sign.detached.verify(messageBytes, sigBytes, pubkeyBytes)` → else `signature_invalid`
4. `publicKey` allowed by current tier (see Allowlist below) → else `not_authorized`

On success: consume the nonce (single-use), set `session.walletAddress = publicKey`, set `session.isOwnerVerified = (publicKey === ownerWallet)`, send `authenticated`.

### 6. Server → `authenticated`

```json
{
  "type": "authenticated",
  "walletAddress": "DwLLgBwG3eVNqHnp9CPsRKL2dVePVdXJq1tvNHCMC2YS",
  "isOwner": true,
  "sessionId": "8f3b1c2a-..."
}
```

Chat-plane messages (`message`, `tx_result`, `tx_error`) are now accepted. The `wallet_connect` and `wallet_disconnect` messages are removed — the wallet is bound at auth time.

### 7. Auth failures

`{type: "auth_error", code: "<reason>", message: "..."}`, then `ws.close(4001, code)`. Codes: `nonce_expired`, `nonce_invalid`, `message_mismatch`, `signature_invalid`, `not_authorized`, `auth_timeout`.

## Allowlist tiers

`AGENT_AUTH_MODE` env var, with smart defaults:

| Mode | Default for | Allowed wallets |
|---|---|---|
| `owner` | `AGENT_MODE=autonomous` | Resolved on-chain asset owner only (existing `resolveOwner`). |
| `allowlist` | `AGENT_MODE=public` if `WALLET_ALLOWLIST` is set | Owner ∪ explicit list. |
| `open` | `AGENT_MODE=public` if `WALLET_ALLOWLIST` is empty | Any valid SIWS signer; per-wallet rate limit applies. |

The owner is **always** authorized regardless of tier (so the owner can manage their own agent without listing themselves). `isOwner: true` is surfaced in `authenticated` and used by `withAuth` for owner-gated tools (unchanged).

### Allowlist source: file (primary) + env (fallback)

The wallet list is loaded from a JSON file with hot-reload, falling back to an env var when no file is present (useful for ephemeral cloud filesystems where mounting a config file is awkward).

- **Primary:** `wallets.allowlist.json` at the workspace root (path overridable via `WALLET_ALLOWLIST_PATH`).
  ```json
  {
    "wallets": [
      "DwLLgBwG3eVNqHnp9CPsRKL2dVePVdXJq1tvNHCMC2YS",
      "AS3yQUgPgsEctYHJ8gJ5xZyL2Nq7kJZ5dq8Hh6BvjMq2"
    ]
  }
  ```
- **Fallback:** `WALLET_ALLOWLIST=pk1,pk2,...` env var. Used when the file is absent. Always merged with the file when both are present (file ∪ env, deduped).
- **Hot-reload:** mtime-poll the file every 5s and reload on change. No process restart needed when the operator adds/removes a wallet. Malformed JSON keeps the last good list and logs a warning.
- **Gitignored:** the file ships as `wallets.allowlist.example.json`; the real file is operator-local.

import nacl from 'tweetnacl';
import bs58 from 'bs58';

export interface VerifySiwsParams {
  message: string;
  signatureBase58: string;
  publicKeyBase58: string;
}

export function verifySiwsSignature(p: VerifySiwsParams): boolean {
  let sig: Uint8Array;
  let pk: Uint8Array;
  try {
    sig = bs58.decode(p.signatureBase58);
    pk = bs58.decode(p.publicKeyBase58);
  } catch {
    return false;
  }
  if (sig.length !== 64 || pk.length !== 32) return false;
  const msgBytes = new TextEncoder().encode(p.message);
  try {
    return nacl.sign.detached.verify(msgBytes, sig, pk);
  } catch {
    return false;
  }
}
```

import { randomBytes } from 'crypto';

export interface IssuedNonce {
  nonce: string;
  issuedAt: string;
  expiresAt: string;
}

export type ConsumeResult =
  | { ok: true }
  | { ok: false; reason: 'nonce_invalid' | 'nonce_expired' };

interface Entry {
  expiresAtMs: number;
}

export interface NonceStoreOptions {
  ttlMs: number;
  now?: () => number;
}

export class NonceStore {
  private readonly entries = new Map<string, Entry>();
  private now: () => number;

  constructor(private readonly opts: NonceStoreOptions) {
    this.now = opts.now ?? (() => Date.now());
  }

  /** Test seam — production callers never use this. */
  setNow(fn: () => number): void {
    this.now = fn;
  }

  issue(): IssuedNonce {
    const nonce = randomBytes(16).toString('hex');
    const issuedMs = this.now();
    const expiresMs = issuedMs + this.opts.ttlMs;
    this.entries.set(nonce, { expiresAtMs: expiresMs });
    return {
      nonce,
      issuedAt: new Date(issuedMs).toISOString(),
      expiresAt: new Date(expiresMs).toISOString(),
    };
  }

  consume(nonce: string): ConsumeResult {
    const entry = this.entries.get(nonce);
    if (!entry) return { ok: false, reason: 'nonce_invalid' };
    this.entries.delete(nonce); // single-use, even on expiry
    if (this.now() > entry.expiresAtMs) {
      return { ok: false, reason: 'nonce_expired' };
    }
    return { ok: true };
  }

  /** Periodic sweep to bound memory under SIWS-flood. */
  sweep(): void {
    const now = this.now();
    for (const [k, v] of this.entries) {
      if (now > v.expiresAtMs) this.entries.delete(k);
    }
  }
}
```

export type AuthMode = 'owner' | 'allowlist' | 'open';

export interface AuthCheck {
  mode: AuthMode;
  publicKey: string;
  owner: string | null;
  allowlist: readonly string[];
}

export function isAuthorized(c: AuthCheck): boolean {
  // Owner is always authorized regardless of mode (so the operator can
  // always reach their own agent without listing themselves).
  if (c.owner !== null && c.publicKey === c.owner) return true;

  switch (c.mode) {
    case 'owner':
      return false;
    case 'allowlist':
      return c.allowlist.includes(c.publicKey);
    case 'open':
      return true;
  }
}
```

import { readFileSync, statSync } from 'node:fs';

export interface AllowlistFileOptions {
  path: string;
  envFallback: readonly string[];
  /** Polling interval for hot-reload, in ms. 0 disables polling. */
  pollIntervalMs?: number;
}

interface FileSnapshot {
  mtimeMs: number;
  wallets: string[];
}

export class AllowlistFile {
  private snapshot: FileSnapshot | null = null;
  private merged: string[] = [];
  private pollTimer: ReturnType<typeof setInterval> | null = null;

  constructor(private readonly opts: AllowlistFileOptions) {
    this.reload();
    if (opts.pollIntervalMs && opts.pollIntervalMs > 0) {
      this.pollTimer = setInterval(() => this.reload(), opts.pollIntervalMs);
      this.pollTimer.unref?.();
    }
  }

  /** Stop the hot-reload poller. Idempotent. */
  stop(): void {
    if (this.pollTimer) {
      clearInterval(this.pollTimer);
      this.pollTimer = null;
    }
  }

  /** Force a re-read (test seam + manual reload). */
  reload(): void {
    let fileWallets: string[] = this.snapshot?.wallets ?? [];
    try {
      const stat = statSync(this.opts.path);
      if (!this.snapshot || stat.mtimeMs !== this.snapshot.mtimeMs) {
        const raw = readFileSync(this.opts.path, 'utf8');
        const parsed = JSON.parse(raw) as { wallets?: unknown };
        if (!Array.isArray(parsed.wallets) || !parsed.wallets.every((w) => typeof w === 'string')) {
          throw new Error('expected { "wallets": string[] }');
        }
        fileWallets = parsed.wallets as string[];
        this.snapshot = { mtimeMs: stat.mtimeMs, wallets: fileWallets };
      }
    } catch (err) {
      const code = (err as NodeJS.ErrnoException).code;
      if (code === 'ENOENT') {
        // File missing — fall back to env only.
        this.snapshot = null;
        fileWallets = [];
      } else {
        // Malformed file or transient read error — keep last good list.
        console.warn(
          `[allowlist] failed to reload ${this.opts.path}: ${err instanceof Error ? err.message : String(err)}; keeping previous list`,
        );
        return;
      }
    }
    const merged = new Set<string>([...fileWallets, ...this.opts.envFallback]);
    this.merged = [...merged];
  }

  current(): readonly string[] {
    return this.merged;
  }
}
```

private async handleAuthResponse(session: Session, msg: ClientAuthResponse): Promise<void> {
  const config = getConfig();
  if (!session.pendingNonce) {
    return this.failAuth(session, 'nonce_invalid', 'No pending nonce.');
  }
  const consumed = this.nonceStore.consume(session.pendingNonce);
  session.pendingNonce = null;
  if (!consumed.ok) {
    return this.failAuth(session, consumed.reason, 'Nonce rejected.');
  }

  // Re-derive the canonical message from the server's known parameters.
  // We can't recover issuedAt/expiresAt from the consumed nonce, so we
  // accept the client-supplied `message` BUT verify it parses to the
  // exact nonce we just consumed.
  if (typeof msg.message !== 'string' || !msg.message.includes(`Nonce: ${this.lastConsumedNonce(msg)}`)) {
    return this.failAuth(session, 'message_mismatch', 'Signed message does not match challenge.');
  }
  // (Implementation note: store {nonce → {issuedAt, expiresAt, agentName...}}
  //  in NonceStore so we can re-derive the canonical string and compare
  //  byte-for-byte. See refinement in Step 4b.)

  if (!verifySiwsSignature({
    message: msg.message,
    signatureBase58: msg.signature,
    publicKeyBase58: msg.publicKey,
  })) {
    return this.failAuth(session, 'signature_invalid', 'Bad signature.');
  }

  const owner = await resolveOwner(config.AGENT_ASSET_ADDRESS ?? null);
  const allowed = isAuthorized({
    mode: config.AGENT_AUTH_MODE,
    publicKey: msg.publicKey,
    owner,
    allowlist: config.WALLET_ALLOWLIST,
  });
  if (!allowed) {
    return this.failAuth(session, 'not_authorized', 'Wallet not on allowlist.');
  }

  session.authStatus = 'authenticated';
  session.walletAddress = msg.publicKey;
  session.isOwnerVerified = owner !== null && msg.publicKey === owner;
  if (session.authTimeout) {
    clearTimeout(session.authTimeout);
    session.authTimeout = null;
  }
  session.send({
    type: 'authenticated',
    walletAddress: msg.publicKey,
    isOwner: session.isOwnerVerified,
    sessionId: session.id,
  });
  this.emitContext(session);
}

```typescript
export interface WalletRateLimiterOptions {
  max: number;
  windowMs: number;
  maxKeys: number;
  now?: () => number;
  ownerExempt?: string | null;
}

interface Entry {
  timestamps: number[];
  lastTouchedMs: number;
}

export class WalletRateLimiter {
  private readonly entries = new Map<string, Entry>();
  private readonly now: () => number;

  constructor(private readonly opts: WalletRateLimiterOptions) {
    this.now = opts.now ?? (() => Date.now());
  }

  allow(pubkey: string): boolean {
    if (this.opts.ownerExempt && pubkey === this.opts.ownerExempt) return true;

    const now = this.now();
    let entry = this.entries.get(pubkey);
    if (entry) {
      // Slide the window.
      entry.timestamps = entry.timestamps.filter((t) => now - t < this.opts.windowMs);
      // Touch order: re-insert to maintain Map insertion-order = recency.
      this.entries.delete(pubkey);
      entry.lastTouchedMs = now;
      this.entries.set(pubkey, entry);
    } else {
      entry = { timestamps: [], lastTouchedMs: now };
      // Evict the oldest entry if at capacity (Map iteration order = insertion).
      if (this.entries.size >= this.opts.maxKeys) {
        const oldestKey = this.entries.keys().next().value;
        if (oldestKey !== undefined) this.entries.delete(oldestKey);
      }
      this.entries.set(pubkey, entry);
    }

    if (entry.timestamps.length >= this.opts.max) return false;
    entry.timestamps.push(now);
    return true;
  }

  /** Test seam — total tracked pubkeys. */
  size(): number {
    return this.entries.size;
  }
}
```

## Resolved design decisions

1. **Allowlist source: file-based with env fallback.** Primary source is `wallets.allowlist.json` at the workspace root (path overridable via `WALLET_ALLOWLIST_PATH`), polled every 5s for hot-reload. `WALLET_ALLOWLIST` env var is merged in as a fallback for cloud deploys without a writable filesystem. File is gitignored; ships as `wallets.allowlist.example.json`. (Task 5.)
2. **Per-wallet rate limit: 60 messages per 60 seconds, owner-exempt.** Sliding window keyed by authenticated pubkey, applied across all of that wallet's concurrent sessions. LRU-bounded at 10,000 distinct wallets to prevent memory flood. Tunable via `WALLET_RATE_LIMIT_MAX` / `WALLET_RATE_LIMIT_WINDOW_MS`. (Task 10.)
3. **No long-lived session JWT.** Each WebSocket performs its own SIWS handshake on connect. Reconnecting = re-signing one short message. Revisit if UX feedback demands a "remember me" mode.
